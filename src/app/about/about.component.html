Features

Multiple lists

Each unique dbkey is it's own encrypted todo list. Create a new list just by changing the dbkey on the homepage or in the URL. Pick a unique DbKey, perhaps your email address plus some random letters, or the current date and time. It doesn't matter what you choose, as long as it is unique and un-guessable.

GTD Compatible

Use @ for Contexts, + for Projects, t:[date] for tickler (threshold) dates. Group by project, context or threshold.

Completed tasks

Show, hide or remove completed tasks. Completed items show up struck-out.

Fully encrypted

DBKey is hashed using SHA1, for the database key. The data is encrypted using AES using a key generated from the DBKey and a salt using PBKDF2. HTTPS for security between the server and browser.

Bookmark your list, if you lose/forget your DBKey, your data is unrecoverable.

Todo.txt Compatible

Uses the Todo.txt specification for text based todo items. Import/export from other Todo.txt compatible task lists. Group by Priority, Project, Context, Due date, Completion date.

Priorities

Use (A), (B), (C) ... (Z), for priorities, also group by priority in priority order. Priority items show up in bold.

Threshold/tickler dates

Set a date when the todo item will showup in the todo list. t:2022-01-01. Show or hide future items.

Past threshold dates show up in dark red, threshold today show up in dark yellow.

Due dates

Items due today, show up in yellow, past due items show up in red, due tomorrow show up in blue. due:2022-01-01

Dates

Supports multiple date formats when saving/updating items. The non-formal date will be replaced with the calculated date.

  • Use yesterday, today, tomorrow
  • Upcoming day of the week - sun...sat or sunday...saturday
  • Relative dates - 2d, 1m, 1y - use days, weeks, months, or years.
  • Partial Dates - No spaces, Oct-4, Dec-25, or Jan-1.
  • Full Dates - 2022-10-04, Oct-4-2022

Recurring items

Strict or weak recurring tasks. rec:+1w, when completed will create a new item 1 week from the current due or threshold date. rec:1w will create a new item 1 week from today. Supported periods d - day(s), w - week(s), m - month(s), y - year(s).

Examples:

  • Weekly review every sunday t:sunday rec:+1w
  • Clean cat's litterbox weekly t:sunday rec:1w

Grouping

Show no groupings (default), or group by: Context, Project, Priority, Due date, or Threshold date.

Filtering

Filter tasks by text, one per line. Filters are ANDed together. Exclude text with a "-" sign. I.e. -rec:.

Add multiple items

Multiple items can be added or uploaded at the same time, one per line.

Item chaining

Do something to:"Do something else later" will create an item Do something else later when the primary item is completed.

Keyboard shortcuts

Shift+Ctrl+U will reload the current page's data.
